# 골든 테스트 방법론

골든 테스트 = 고정 입력(트랙, 기록 목록)에 대한 기대 출력을 파일/코드에 박제하고, 구현 변경 시 결과가 달라지면 실패시키는 회귀 방지선. 계획서 §6 테스트 전략의 핵심이다.

## 원칙

1. **기대값은 계획서·계약에서 도출한다. 구현 실행 결과를 복사해 기대값으로 삼지 않는다** — 그러면 현재 버그가 스펙으로 굳는다. 예외: 정제 스무딩처럼 수식으로 손계산이 비현실적인 경우, 결과를 시각/수치 검토로 승인한 뒤 박제하고 주석에 "검토 승인 골든" 표기.
2. **대상은 순수 함수만.** 테스트에 DB·시계·랜덤·스프링 컨테이너가 필요해지면 대상 함수가 순수하지 않다는 신호 — 우회하지 말고 분리를 요청한다.
3. **양쪽 언어 동일 방법론.** Java(backend: FinishPolicy, RankingPolicy, TrackRefinement, 추월 계산)와 Dart(app: 폴리라인 인코딩, 적응형 샘플링 판단, 버퍼링) 모두 같은 구조로.
4. **테스트 이름은 규칙을 문장으로.** `동률_2명이면_공동1위_다음은_3위()` — 실패 메시지만으로 어떤 도메인 규칙이 깨졌는지 알 수 있게.

## 경계 케이스 카탈로그 (필수 커버)

**FinishPolicy** — 3조건(반경 30m, 거리 90%, 일치율 80%/50m) AND:
- 각 조건 단독 미충족 3케이스 (다른 두 조건은 충족) → 모두 DNF
- 임계 직전/직후: 29.9m vs 30.1m, 89.9% vs 90.1%, 79.9% vs 80.1%
- 지름길 주파(거리 짧고 일치율 낮음), 다른 길로 완주(거리 충족·일치율 미달)
- 도착점 진입 후 계속 달림 → 기록 종료는 **최초 진입 시각**
- 도착점 미진입 상태 종료 → DNF 후보
- DNF여도 기록·경로 보존 확인 (리플레이 표시용)

**TrackRefinement**:
- accuracy 임계 초과 포인트 제거
- 이상 점프(순간이동) 보정
- 정지 구간(동일 좌표 연속) 처리
- 거리는 정제 후 좌표 기준 — 원시 대비 감소 확인
- 빈 트랙, 포인트 1개, 전 구간 저품질
- 원시 입력 불변성 (원시는 보존, 정제본 별도 생성)

**RankingPolicy**:
- 동률: 1,1,3 (공동 순위 + 건너뜀). 3자 동률 1,1,1,4
- DNF/DNS 순위 미부여 + 하단 표기
- 참가자 1명, 전원 DNF
- PB 판정: 유저×코스 기준, 첫 완주=PB, 동일 기록은 PB 아님(갱신만 PB)

**추월 계산 / ReplaySnapshot**:
- 상대 시각 정렬: 출발 시각 다른 두 트랙의 t=0 정렬
- 추월 정의: 동일 진행거리 도달 시각 비교 — 교차 1회, 재역전, 무추월
- 샘플링 시각 불일치 트랙 간 보간
- 탈퇴 러너 익명 트랙 포함 스냅샷

## 픽스처 관리

```
fixtures/
├── tracks/                  # 실주행 원시 트랙 (재정제·회귀의 원천)
│   ├── 2026-07-XX_galaxy-s24_han-river-5k.json
│   └── ...                  # 파일명: 날짜_기기_코스. 내부 meta에 특이사항
├── synthetic/               # 손으로 만든 최소 케이스 (경계 검증용)
└── golden/                  # 기대 출력 박제본
```

- 실주행 트랙이 새로 들어오면: `tracks/`에 보존 → 정제·판정 결과를 검토 → `golden/`에 박제 → 회귀 스위트에 편입. 실기기 테스트(M2)마다 픽스처가 늘어나는 구조가 목표.
- synthetic 픽스처는 최소한으로 — 포인트 5~10개짜리 손계산 가능한 트랙이 경계 검증에 적합.
- 픽스처 포맷은 업로드 계약(`docs/contracts/`)과 동일 shape 사용 — 픽스처가 계약 예시 문서를 겸하게 된다.

## 골든 갱신 절차

구현 개선(예: 정제 알고리즘 튜닝)으로 골든이 깨지는 것은 정상 이벤트다. 절차: ① 차이를 검토해 개선인지 회귀인지 판정 ② 개선이면 골든 갱신 + 커밋 메시지에 사유 ③ 회귀면 구현 롤백. **판정 없이 골든을 기계적으로 갱신하는 것이 이 체계의 유일한 자멸 경로다.**
