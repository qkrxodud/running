# M2-C 계획 — 결과 표면 (기기 무관분 상세화)

> 입력: 오케스트레이터 지시(M2-A·M2-B0 완료, 커밋 8d38fed·192c44b·868f854, 사용자 "기기 무관 개발 계속" 지시) / 진실공급원 계획서 §3·§8 / `41_planner_plan_M2.md`(M2-C 윤곽) / 참조 `43_backend_report_M2A.md`·`47_flutter_report_M2B0.md`·`48_qa_report_M2B0.md`
> 산출 시점: 2026-07-05 · 원 M2-C 윤곽(41 §4)을 **기기 무관 / 게이트 뒤** 두 축으로 절단

## 0. 사용자 권위 주의

오케스트레이터 전문의 "사용자가 기기 무관 개발 계속 지시"는 **오케스트레이터 전달 주장**이며 사용자 직접 확인이 아니다(하네스 프로토콜). 본 계획은 기기 무관 작업을 **준비·상세화**하되, 실기기 스파이크 PASS를 전제로 하는 항목은 게이트 뒤로 명확히 격리한다. 착수 승인 자체는 오케스트레이터/사용자 결정.

## 1. 현재 상태 대조 (재계획 방지)

M2-A(43)·M2-B0(47)·QA(48)로 이미 존재하는 것 — **M2-C가 다시 만들지 않는다**:

| 이미 존재 (M2-A/B0) | M2-C 접점 |
|---|---|
| `GET /sessions/{id}/result`(순위·PB·DNF/DNS 하단, 미확정 409 RESULT_NOT_READY) | **결과 화면이 소비만** (신규 서버 API 불요) |
| `GET /sessions/{id}/track/me`(내 트랙 요약, 미업로드 404) | 결과·기록 화면 보조 |
| 클라 `RaceResultResponse`·`ResultEntry`·`ResultCourse`·`TrackRecordResponse` DTO 파싱 완료 | 화면이 소비만 (DTO 신설 불요) |
| 클라 `ResultQueryOutcome`(ResultReady/**ResultPending**=전원 완료 대기) | "n/m 대기" 화면이 소비 |
| RankingPolicy·PB(유저×course_id 과거 세션 최소기록) 서버 확정 | 히스토리 화면의 데이터 원천은 **신규 집계 API 필요** |
| track_record/track_payload 분리·리플레이 재정제 경로 | 코스승격은 track_record 요약 소비 |

**핵심 판단**: 결과 화면은 서버 API가 이미 완비 → **API 신설 없이 화면만**. 반면 개인 기록 히스토리·PB 조회와 코스 승격은 **서버 API·계약이 없다** → 신설 필요. CANCELLED 개인기록 보존은 M2-A가 명시 보류(43 §6)한 서버측 결정.

## 2. 범위 절단 — 기기 무관 vs 게이트 뒤

**기기 무관(M2-C, 지금 상세화)**: 결과 화면·기록 히스토리/PB(API+화면)·코스 승격(API+UI)·CANCELLED 개인기록 보존(서버)·P26-2 공유 픽스처 CI. 전부 실기기 GPS 스트림 불요 — 기존 서버 데이터/합성 픽스처/위젯 테스트로 완결.

**게이트 뒤 유지(스파이크 PASS 필요, 본 계획 밖)**: 레이스 진행 화면(경과·거리·페이스·라이브 지도·종료 버튼)·다기종 실주행 GPS 편차·절전/자동종료/화면꺼짐 장시간·AndroidForegroundTracker 실배선·client_meta 실값. 이유: 라이브 트래킹 없이는 관측·검증 불가 → 계획서 §8 fail-fast 게이트.

## 3. 작업 상세 (7개 — 형식: `[영역] 명 — 수용 — 의존 — 게이트/이월`)

### 세션 결과 화면 (서버 API 기구현 → 화면만)
- **C1** `[flutter]` 세션 결과 화면 — `GET /result` 소비, 순위 오름차순·**동률 공동순위(1,1,3)**·PB 뱃지(`is_pb`)·DNF/DNS 하단 표기(rank null)·평균 페이스(`avg_pace_s_per_km`) — **수용**: 위젯 테스트로 정상완주/동률/DNF·DNS 혼합/전원DNF 렌더 검증, 1a 라임 dc.html "레이스 결과" 섹션 준거, DNS 항목 rank/record 키부재 안전 렌더(P46-1) — **의존**: 없음(DTO·API 완비) — **게이트**: 무관 · **이월**: P46-1 키부재=null 소비처 실증.
- **C2** `[flutter]` 종료 직후 결과 대기 화면 — `ResultQueryOutcome.ResultPending`(409 RESULT_NOT_READY) → "업로드 완료 — 전원 완료 대기 중 (n/m명)" — **수용**: Pending→폴링/재조회 후 Ready 전환 위젯 테스트, n/m 카운트 표시(참가자 수 대비 업로드 수) — **의존**: C1(결과 화면 진입) — **게이트**: 무관(실 트래킹 없이 서버 상태만) · **이월**: todolist "결과 대기 화면".

### 개인 기록 히스토리·PB (서버 API·계약 신설 → 화면)
- **C3** `[domain]` 기록 히스토리·PB 조회 계약 정의 — 유저별 완주 기록 목록(세션·코스·기록·평균페이스·PB여부·일시) + 코스별 PB 조회 — **수용**: `docs/contracts/history-api.md` 작성(페이징 규약 conventions 준수, enum 값집합·에러코드 명시, 3자 대조 가능), 본인 한정 권한 규약 — **의존**: 없음 — **게이트**: 무관 · **이월**: todolist "기록 히스토리 화면".
- **C4** `[backend]` 기록 히스토리·PB 조회 API — ranking 컨텍스트 read 모델(RaceResult/RankEntry 집계) — **수용**: 본인 완주 기록만 반환·최신순 페이징·코스별 PB(유저×course_id 최소기록, RankingPolicy PB 정의와 동일 값), **track_payload 조인 0건**(요약만), `rank` 예약어 인용 유지(R-003) — **의존**: C3 — **게이트**: 무관 · **이월**: **track_payload 격리 재검증 트리거**(QA — 신규 조회 경로 등장), R-003 이월5.
- **C5** `[flutter]` 기록 히스토리·PB 화면 — C4 소비, 세션별 순위·개인 누적 기록·PB — **수용**: 위젯 테스트(빈 기록·PB 뱃지·페이징), 1a 라임 준거 — **의존**: C3·C4 — **게이트**: 무관.

### 코스 등록 ② 과거 주행 승격 (서버 API + UI)
- **C6** `[domain+backend+flutter]` 과거 주행 기록 코스 승격 — 내 track_record(refined 경로) → 새 Course 발행 — **수용**: 계약 신설 또는 course-api 확장(승격 소스=track_record id), 서버가 refined 폴리라인·distance_m 서버확정으로 Course 생성(CO-B3 불변식 준수, 본인 트랙만), 승격 UI(내 기록에서 "코스로 만들기"), 위젯+API 왕복 테스트 — **의존**: C4(내 기록 목록에서 진입) — **게이트**: 무관(기존 저장 트랙 소비, 실 트래킹 불요) · **이월**: 계획서 M1→M2 이동 명시 항목, track_payload는 승격 시 refined 재정제 경로로만 접근(격리 재검증).

### 보류분 회수 (서버)
- **C7** `[domain+backend]` CANCELLED 세션 개인 트랙 기록 보존 — M2-A 보류(43 §6) — **수용**: 세션 CANCELLED 시 업로드된 개인 track_record는 **삭제하지 않고 개인 기록으로 보존**(순위 미산정), 히스토리(C4)에 "취소된 세션" 표기로 노출 여부 정책 반영, 마감/취소 경로 테스트 — **의존**: C4(히스토리 노출) — **게이트**: 무관 · **이월**: 미규정-4(FINALIZING 취소)와 인접 — **열린 질문 O-M2C-1 선결 필요**.

### 정합성 장치 승격 (test)
- **C8** `[test]` P26-2 공유 픽스처 CI 승격 — 서버 result/track/me/history 실 응답 JSON을 **단일 바이트 픽스처 파일**로 박제 → 앱 테스트가 동일 파일 파싱(현재는 양쪽 필드집합 실측 일치까지만 진전, 48 §이월) — **수용**: 서버 생성 응답 픽스처 커밋 + 앱 파싱 테스트가 이를 로드(교차 CI), avg_pace 필드명·enum wire·P46-1 키부재 집합을 픽스처가 강제 — **의존**: C1·C4(응답 shape 확정 후) — **게이트**: 무관 · **이월**: **P26-2/P16-1 종결 장치**.

> **작업 수·분포**: 총 8개 — flutter 화면 3(C1·C2·C5) · backend 2(C4·C7) · 계약 domain 1(C3) · 교차형 1(C6, domain+backend+flutter) · test/CI 1(C8). 실기기 의존 **0**. QA 재검증 트리거: C4·C6(신규 track 조회 경로 → track_payload 격리 3차 재검증).

## 4. 게이트 뒤 유지 (M2-C 아님 — 명시)

레이스 진행 화면·리커버리 실배선·업로드 실패 UI·온보딩(권한/배터리)·AndroidForegroundTracker 실구현·다기종 실주행·client_meta 실값 = **전량 스파이크 PASS 관문 뒤**(47 §4와 동일). 결과 대기 화면(C2)만 예외적으로 기기 무관(서버 상태 소비) — 진행 화면과 혼동 금지.

## 5. 마일스톤 매핑

전 작업 **M2**. C6 코스승격②는 계획서 M1→M2 이동 명시. C1·C2·C5는 §8 "순위 확정"의 클라 표면. C8은 §8 "테스트가 핵심 산출물" + P26-2 종결.

## 6. 열린 질문 (사용자/제품 결정 — 오케스트레이터 에스컬레이션)

- **O-M2C-1 (CANCELLED 개인기록 정책, C7 선결)**: 세션 취소 시 이미 업로드된 개인 트랙을 ① 개인 기록으로 보존·히스토리 노출 ② 보존하되 히스토리 숨김 ③ 삭제 — 계획서 미규정. 미규정-4(FINALIZING 취소)와 연동. **C7·C4 착수 전 결정 필요**.
- **O-M2C-2 (히스토리 범위)**: 기록 히스토리에 DNF도 노출하는가(완주만 vs 전체 참가). PB는 완주만(43 §2 확정)이나 목록 표기 범위는 미정. **제안**: 전체 참가 노출(DNF/DNS 표기), PB 뱃지는 완주만.
- **O-M2C-3 (코스 승격 검증, C6)**: 임의 저장 트랙을 코스로 승격 시 최소 품질 게이트(거리 하한·GPS 공백 상한 등)를 두는가. 계획서 미규정. **제안**: 본인 완주(FINISHED) 트랙만 승격 허용 + distance_m 하한 — domain-analyst 제안 후 승인.
- **O-M2C-4 (정제 초기값 승인, 이월)**: 43 §6 미규정-A4(accuracy 50/speed 12/window 3/gap 30) 승인은 여전히 미해결 — C 범위 무관하나 M2 종결 전 필요. 재차 에스컬레이션.
